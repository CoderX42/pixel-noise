# PixelNoise UI/UX 设计提示词

将下列内容作为**视觉与交互设计**的约束与灵感来源，用于指导 PixelNoise 类产品或同类像素风白噪音页面的设计与还原。

---

## 设计定位

- **风格锚点**：沙盒建造类游戏（如 **Minecraft**）的 **HUD / 物品栏 / 按钮方块** 美学，而非扁平 Material 或通用 SaaS 仪表板。
- **气质**：复古 **8-bit / 像素**，可读、可玩、有「方块厚度」与手工感；避免「AI 通用灰白卡片风」。
- **用途**：环境音 / 白噪音 / 专注辅助；界面应**沉浸但不抢眼**，控件一眼可辨、拇指友好。

---

## 视觉系统

### 布局与网格

- 基础对齐：**8px** 或 **4px** 倍数；图标区域可对照 **16×16** 逻辑像素。
- 层次：**全屏 Canvas 场景** 在下；**半透明或渐变 HUD** 叠在边缘（顶栏、底栏、横屏侧栏），不挡主视觉中心过久（如大播放键可在播放后淡出）。

### 色彩

- **石头灰 / 板岩感**：未选中槽位、中性按钮。
- **钻石蓝 / 青**：选中、滑块手柄、强调描边。
- **熔岩橙 / 暖色**：主播放、危险或高关注操作（如定时运行中）。
- **草地绿 / 深空**：可选作日/夜或背景氛围色，与场景主色区分（UI 色与场景色可呼应但不必完全一致）。
- **煤炭黑**：描边、文字描影；所有「方块」需有 **2px 级描边** 与 **顶亮底暗** 的浮雕感。

### 字体

- **标题 / 按钮**：像素风显示字体（如 **Silkscreen**、或规范中的 **Press Start 2P**）。
- **数字 / 标签**：等宽像素正文（如 **VT323**）。
- **禁止**：Inter、Roboto、Arial 等作为视觉主字体。

### 控件形态

- **按钮**：外框高光在左上、阴影在右下；**按下**时高光阴影对调并 **translate(1–2px)** 模拟按下。
- **槽位（Hotbar）**：未选中为「凹进」或中性块；**选中**为亮色描边或整槽提亮，可加 **角标 / 呼吸描边**（`steps` 动画保持像素感）。
- **滑块**：深色轨道 + 亮色填充 + **方块手柄**（中间一道高光竖线）；拖动时可略微加宽或变色。
- **Tooltip**：小面板 + 对比色描边，**VT323** 说明文案；悬浮显示场景名 + 一句氛围描述。

### 图标

- **16×16 网格** 手绘矩形拼像素图标（雨、林、浪、火、雪、噪），与场景一一对应；优先 **SVG rect** 或 Canvas 绘制，避免糊边。

---

## 动效与交互（UX）

### 原则

- **少而准**：动效服务反馈（按下、选中、播放中），避免满屏飘动干扰专注。
- **像素感**：过渡可用 **`steps()`** 关键帧，模拟逐帧游戏；慎用长时间平滑 blur。

### 建议动效

- **场景切换**：全屏或覆盖层 **像素块随机溶解**（先盖再揭）。
- **播放中**：主播放键 **轻微脉冲**（缩放放在**内层**，避免触摸区域错位）。
- **Logo / 标题**：极慢 **text-shadow 或光晕色偏**，不抢戏。
- **底部栏**：随当前场景 **渐变色调** 微弱变化，强化「身在景中」。
- **画布**：点击 **方形涟漪**；涟漪颜色随场景主色变化。

### 移动端 UX

- 触摸目标 **≥ 44px**；**`touch-action: manipulation`**；适当 **`-webkit-tap-highlight`**。
- **播放 / 暂停** 同时提供 **中央大键 + 底栏小键**；中央键可在播放稳定后自动隐藏，减少遮挡。
- 横屏：**侧栏竖向槽位** + **底栏控制条**，与竖屏 **横向 Hotbar** 行为一致。

### 无障碍与反馈

- 按钮 **`type="button"`**、`aria-label`（播放/暂停）；隐藏装饰元素 **`aria-hidden`**。
- 首次播放可 **`toast`** 提示「已开始」（一次性），符合浏览器自动播放策略下的用户预期。

## 文案与语气（中文）

- 简短、游戏感：如「MIX 混音模式」「雨声 · 窗外淅沥」。
- 避免冗长说明挤占像素界面；细节放 tooltip 或 README。

---

## 反模式（不要做）

- 大圆角卡片 + 紫色渐变 + 系统无衬线标题的「通用 AI 落地页」。
- 无描边的扁平按钮、无按下态的控件。
- 场景与听感严重不符（如「海浪」只有静态粉噪无起伏）。
- 手机仅依赖 `click`、无去重，导致双击误触或点不动。
